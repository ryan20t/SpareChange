Week 6 Update

This week I have done a lot of work on the settings page. Income information is modifiable and displayed. Bills are now displayed. It is mostly functioning, with the noticeable exception of the savings goal not being implemented yet. I also spent some time trying to switch the SQL code over to stored procedures, only to realize that PhP is not terribly friendly with stored procedures. This project was scrapped. Additionally, I added a confirmation prompt before the deletion of a bill or transaction.

The application is not hosted live. It can be found here: http://sparechange.comoj.com/login.php

I’ve set up a website detailing the project and my progress. It can be viewed here: http://ryan20t.github.io/SpareChange/

Next week the goal will be to finish up the settings page, and include the savings goal meter on the dashboard to the working sections of the application.

Week 5 Update

The development phase of the project has begun. The sign up and log in functions are essentially complete. Transaction insertion is functional, and the today view section of the dashboard is mostly finished with the exception of the savings goal meter, which will be completed after the ability to create a goal is coded. Today’s transactions are displayed, as is daily spending limit, amount spent, and amount remaining.

The design of the settings page has been reworked to better represent its contents. Insertion of new static and budgeted bills is functional. The bills are not yet displayed back to the user, but this will be the next thing I add.

Error handling has been added for sign up, log in and bill entry, and next to transaction entry. It aggregates input errors and can output them, but is not fully implemented yet. I need to incorporate the displayed messages into the design of the application.
